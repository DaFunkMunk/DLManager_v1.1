Dynamic Expression Quick Guide

Use this reference to build attribute-based rules with confidence—no coding experience required.

When to Use Expressions

Expressions let you describe membership logic with attribute comparisons instead of static lists. They’re ideal when you need to combine two or more fields—for example, “Contractors in the Permian office” or “Active employees with the Responder tag”.

Syntax Essentials

  • Use lowercase logical operators: and, or, not.
  • Wrap text values in double quotes: location == "Houston HQ".
  • Numbers (such as tenureDays) do not need quotes.
  • Use contains(listField, "value") for array fields like tags or directoryGroups.
  • Expressions are case-sensitive—match the field names exactly as shown below.

Available Fields

  • employmentType
  • location
  • role
  • department
  • manager
  • tags
  • directoryGroups
  • tenureDays
  • orgUnit
  • active (true/false)

Operators & Helpers

  • ==, != — equals / does not equal
  • >, >=, <, <= — numeric comparisons (use with tenureDays)
  • contains(list, value) — checks whether a value exists inside an array field (tags, directoryGroups)
  • not — negates a condition (not active)
  • Group conditions with parentheses: (A and B) or C

Ready-to-Use Snippets

Contractors in Permian Field Office

employmentType == "Contractor" and location == "Permian Field Office"

Responders Who Report to Alex Rivera

contains(tags, "Responder") and manager == "Alex Rivera"

Ops Operators with 1+ Year Tenure

contains(directoryGroups, "DL_Permian_Operators") and tenureDays >= 365

Active Employees Outside Houston

active and location != "Houston HQ"

Troubleshooting

  • Invalid syntax: check for uppercase AND/OR, missing quotes, or unmatched parentheses.
  • No matches: simplify the expression to a single condition to confirm each part returns results.
  • Array fields: always use contains for tags and directoryGroups; equality will not work.
